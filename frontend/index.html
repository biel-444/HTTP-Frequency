Sobre o Projeto

O HTTPFrequency foi criado para facilitar testes HTTP e monitoramento de endpoints em uma interface moderna e intuitiva.

O objetivo é fornecer uma plataforma para execução concorrente de requisições HTTP, medição de latência com a biblioteca HTTPX e geração de relatórios.

Como funciona

01

Entrada de URLs

Insira uma ou mais URLs no campo de texto, uma por linha. O sistema aceita endereços com ou sem https:// e remove duplicatas automaticamente.

02

Concorrência

Define quantas requisições são disparadas ao mesmo tempo. O valor padrão é 10 — aumente para testar mais URLs em paralelo, ou diminua para não sobrecarregar servidores mais lentos.

03

Timeout

Tempo máximo em segundos que o sistema aguarda a resposta de cada URL. O padrão é 10s — reduza para identificar sites lentos mais rapidamente, ou aumente para dar mais tempo a servidores instáveis.

04

Coleta de Métricas

Para cada URL são medidos: tempo de resposta, código HTTP, bytes recebidos e URL final após redirecionamentos.

05

Ranking e Relatório

Os resultados são ordenados por tempo de resposta, gerando um ranking das URLs mais rápidas. Os dados ficam disponíveis para download em CSV.

Sim. O sistema tem histórico persistente integrado ao backend — os dados ficam disponíveis para download em CSV.

Sim. Autenticação e contas de usuários estão planejadas para futuras versões.
